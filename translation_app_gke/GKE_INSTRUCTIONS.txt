GKE DEPLOYMENT INSTRUCTIONS FOR TRANSLATION APP VLLM
====================================================

This guide explains how to build the Docker image, push it to Google Container Registry (GCR), and deploy the application to Google Kubernetes Engine (GKE) Autopilot.

PREREQUISITES
-------------
1.  **Google Cloud Project**: You must have an active GCP project (e.g., `gen-lang-client-0088002625`).
2.  **GKE Cluster**: A GKE Autopilot cluster is recommended.
3.  **Tools Installed on Windows**:
    - `gcloud` CLI: [Install Link](https://cloud.google.com/sdk/docs/install)
    - `kubectl`: (Installed via gcloud components or Docker Desktop)
    - `docker`: Docker Desktop

STEP 1: ONE-TIME SETUP & AUTHENTICATION
---------------------------------------
If you are running this for the first time on a new machine, perform these steps:

1.  **Login to Google Cloud**:
    ```powershell
    gcloud auth login
    ```

2.  **Configure Docker to use Google Credentials**:
    ```powershell
    gcloud auth configure-docker
    ```

3.  **Install GKE Auth Plugin** (Required for kubectl on newer GKE versions):
    ```powershell
    gcloud components install gke-gcloud-auth-plugin
    ```

4.  **Connect to Your Cluster**:
    Replace `CLUSTER_NAME` and `ZONE` with your details.
    ```powershell
    gcloud container clusters get-credentials CLUSTER_NAME --zone ZONE --project gen-lang-client-0088002625
    ```

5.  **Create Hugging Face Secret**:
    Required for downloading the Gemma model.
    ```powershell
    kubectl create secret generic hf-secret --from-literal=token=YOUR_HF_TOKEN
    ```

STEP 2: BUILD AND PUSH IMAGE
----------------------------
Use the provided PowerShell script to build and push the image to GCR.

1.  Open PowerShell in `translation_app_gke`.
2.  Run the script:
    ```powershell
    .\build_and_push.ps1 -ProjectId "gen-lang-client-0088002625"
    ```
    *This pushes to: `gcr.io/gen-lang-client-0088002625/translation-app-gke:latest`*

STEP 3: DEPLOY TO GKE
---------------------
The Kubernetes manifests in `k8s/` are configured for GKE Autopilot (using `nvidia-l4` GPUs).

1.  **Deploy the Models and App**:
    ```powershell
    kubectl apply -f k8s/01-vllm-text.yaml
    kubectl apply -f k8s/02-vllm-ocr.yaml
    kubectl apply -f k8s/03-app.yaml
    ```

2.  **Monitor Deployment**:
    It may take 5-10 minutes for "Pending" pods to start as they need to provision nodes and download large models.
    ```powershell
    kubectl get pods --watch
    ```

3.  **Access the App**:
    When all pods are `Running`, get the External IP:
    ```powershell
    kubectl get svc translation-app-service
    ```
    Open the `EXTERNAL-IP` in your web browser.

TROUBLESHOOTING
---------------
*   **"Permission denied on locations/..."**: Run `gcloud auth login`.
*   **"gke-gcloud-auth-plugin not found"**: Run `gcloud components install gke-gcloud-auth-plugin`.
*   **"Unauthenticated request" during push**: Run `gcloud auth configure-docker`.
*   **"GKE Warden constraints violations"**: Ensure your `.yaml` files have the correct `nodeSelector` (e.g., `cloud.google.com/gke-accelerator: nvidia-l4`).
